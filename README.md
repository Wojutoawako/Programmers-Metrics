# Метрики программистов

This project was generated using [Angular CLI](https://github.com/angular/angular-cli) version 21.2.1.

## Требования по ТЗ:   
- Интеграция с GitLab  
- Возможность показа ключевых параметров программиста: количество коммитов, языки программирования, количество строк кода добавленных/удаленных, количество код-ревью, объем проведенных код ревью  
- Возможность сбора статистики: самый активный день в неделе, самое активное время совершения коммитов
- Возможность сравнения двух и более программистов по ключевым параметрам

## Примерное описание работ:
### Страницы приложения  
#### Страница авторизации  
Простая страница с кнопкой "Войти через GitLab" (Oauth 2) или полем для ввода Personal Access Token. Далее токен сохраняется в приложении. Как минимум - сохранять в открытом виде в localStorage, но лучше подумать о безопасности.  
После успешной авторизации происходит редирект на главную.

#### Главная страница (Dashboard)
Отображает список программистов, доступных для анализа. Это могут быть участники группы GitLab или заранее заданные пользователи (например, из конфигурации).  
Каждый элемент списка содержит: аватар, имя, username, краткую статистику (коммиты за неделю).  
Кнопка "Добавить в сравнение" для добавления программиста в сравнительный анализ.
Кнопка "Детали" для перехода к детальной странице программиста.
Сверху – фильтр по имени и кнопка "Перейти к сравнению".

Дизайн:
Используется карточная сетка Angular Material (mat-card).  

#### Детальная страница программиста
Отображает подробную информацию о выбранном программисте.  
Верхняя часть: аватар, имя, email, общая статистика (всего коммитов, всего MR, общее изменение строк).
Далее – вкладки:
- Коммиты: таблица последних коммитов с датой, сообщением, ссылкой на проект.
- Языки: круговая диаграмма распределения языков программирования (по числу коммитов или изменению строк).
- Активность: два графика – активность по дням недели (столбчатая диаграмма) и активность по часам (линейная/столбчатая).
- Код-ревью: список MR, где программист был ревьюером, с указанием проекта, даты, количества комментариев.

Дизайн:
Вкладки реализованы через mat-tab-group. Графики – через ngx-charts или chart.js. Данные обновляются при смене периода (например, последние 30 дней).

#### Страница сравнения
Достает данные по добавленным в сравнение программистам, также не забываем про плашку, если никого для сравнения не выбрано.  
Также присутствует кнопка "Добавить программиста", которая открывает меню с выбором из текущих программистов.  

Отображает сравнительную таблицу метрик: коммиты, строки, ревью и т.д.

Под таблицей – совмещенные графики: столбчатая диаграмма по коммитам, линейная по активности во времени.

Дизайн:
Используется mat-select для множественного выбора. Таблица – mat-table. Графики – ngx-charts или chart.js.

### Методы для работы с GitLab API
GitLab API предоставляет все необходимые данные. Базовый URL: https://gitlab.com/api/v4/ . Для аутентификации используется токен в заголовке Authorization: Bearer <token>.

#### Получение текущего пользователя
GET /user  
Используется после авторизации для получения информации о владельце токена.

#### Получение списка проектов пользователя
GET /users/:user_id/projects  
Параметры: membership=true (только проекты, где пользователь участник).

#### Получение коммитов пользователя в проекте
GET /projects/:id/repository/commits  
Параметры: author=<username>, since, until, per_page, page.  
Возвращает массив коммитов с метаданными: id, message, committed_date, stats (добавлено/удалено).

#### Получение статистики по коммиту
GET /projects/:id/repository/commits/:sha  
В ответе есть поле stats с additions, deletions, total.  

#### Получение Merge Requests, где пользователь является ревьюером
GET /merge_requests?reviewer_id=<user_id>&state=merged  
Или для конкретного проекта:  
GET /projects/:id/merge_requests?reviewer_id=<user_id>  
Возвращает список MR с информацией об изменениях и комментариях.

#### Получение комментариев к MR (для объема ревью)
GET /projects/:id/merge_requests/:iid/notes  
Можно подсчитать количество комментариев (исключая системные).  

#### Определение языков программирования 
GET /projects/:id/languages – возвращает доли языков в проекте.    
Затем можно пропорционально распределить коммиты пользователя по языкам.

#### Получение активности коммитов по времени
GET /projects/:id/repository/commits    
Из данных коммитов (поле committed_date) можно извлечь день недели и час. Агрегировать на клиенте.

|---------|------------------------------------------------------------------------------------------------|

| №2      | Главная страница и детальная страница программистов                                            |
| №3      | Страница сравнения программистов, агрегация статистики                                         |
